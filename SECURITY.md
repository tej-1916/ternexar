# Security Policy

## Safety Philosophy
TERNEXAR is built on the principle of **Safe Local Autonomy**. We believe that terminal AI should be deterministic, auditable, and locally-controlled. 

1. **Local-Only:** No data is sent to external APIs. All inference and safety checks happen on your machine.
2. **Deterministic Risk:** Safety is not an "opinion" generated by an LLM. It is enforced by a rule-based engine that identifies dangerous patterns.
3. **Auditability:** Every safety decision and execution attempt is logged to a local, tamper-evident audit file.

## Reporting a Vulnerability
If you discover a security vulnerability within TERNEXAR (e.g., a bypass of the risk engine or execution gate), please report it responsibly.

*   **Email:** security@ternexar.local (Placeholder - Please use GitHub Issues for now with the 'Security' label)
*   **Response Time:** We aim to acknowledge reports within 48 hours and provide a fix or mitigation strategy within 7 days.

## Out of Scope
*   Vulnerabilities in the underlying OS or terminal emulator.
*   Security issues in the local Ollama instance (please report those to Ollama).
*   User-driven bypasses (e.g., manually disabling safety checks).

Thank you for helping keep TERNEXAR safe.
